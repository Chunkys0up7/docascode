## üîß How to Use

### Using Templates

1. **Choose your template** from the index
2. **Copy the template** to your documentation location
3. **Replace placeholders** with your content
4. **Follow inline guidance** in the template
5. **Submit for review** via pull request

### Example Usage

```bash
# Copy template for a new concept
cp site_docs/contributing/templates/concept-template.md \
   site_docs/concepts/my-new-concept.md

# Edit and replace placeholders
# Then create PR
```

### Naming Files

Follow these conventions:
- Use lowercase with hyphens
- Be descriptive and specific
- Keep names concise (2-5 words)
- Match documentation type patterns

Examples:
- Concepts: `knowledge-graphs.md`
- Tutorials: `tutorial-first-procedure.md`
- How-To: `how-to-deploy-production.md`
- Reference: `api-reference.md`

### Reporting Issues

Use the GitHub issue templates:
1. Go to repository Issues
2. Click "New Issue"
3. Select appropriate template
4. Fill in the structured form
